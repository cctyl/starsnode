
# 加载不了图片问题

放在public下的图片，打包后变成/img/xxx.svg,
但是在android加载时，会变成file:///img/xxx.svg。


## 排除法
- 不是android webview的问题，因为simple-backup的html放在这个app里，图片可以正常加载，所以确定是web项目的问题
- 在index.html用./img/xxx.svg 可以正常加载
- /img 或者 img ，都不行，都是 file:///img/xxx.svg
- ./img不行，会提示打包失败
- 不是文件类型的问题，.svg改为.jpg也不行
- 和simple-backup打包后文件对比，发现那边也是 /img作为路径，打包后也是/img，但是它的可以正常加载
